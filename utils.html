class leabratf.utils.RotatingFileHandlerRelativePath(filename, *args, **kwargs)¶

Extension of the filehandler class that appends the current directory to the inputted filename. This is so the log files can be found relative to this file rather than from wherever the script is run.

leabratf.utils.setup_logging(path_yaml=None, dir_logs=None, default_level=20)¶

Sets up the logging module to make a properly configured logger.

This will go into the logging.yml file in the top level directory, and try to load the logging configuration. If it fails for any reason, it will just use the default configuration. For more details on how the logger will be configured, see the logging.yml file.

Parameters:
  • path_yaml (str or Path, optional) – Path to the yaml file.
  • dir_logs (str or Path, optional) – Path to the log directory.
  • default_level (logging.LEVEL, optional) – Logging level for the default logging setup if the yaml fails.
leabratf.utils.as_list(obj, length=None, tp=None, iter_to_list=True)¶

Force an argument to be a list, optionally of a given length, optionally with all elements cast to a given type if not None.

Parameters:
  • obj (Object) – The obj we want to convert to a list.
  • length (int or None, optional) – Length of new list. Applies if the inputted obj is not an iterable and iter_to_list is false.
  • tp (type, optional) – Type to cast the values inside the list as.
  • iter_to_list (bool, optional) – Determines if we should cast an iterable (not str) obj as a list or to enclose it in one.
Returns:

obj – The object enclosed or cast as a list.
Return type:

list
